Bug Handling SOP

Shared language for reporting, triage, diagnosis, TDD fixes, and root-cause closure — powered by Matt Cursor skills. Five-Module Course ≈ 2.5–3.5 hours (workshop ≈ 3h).

Why

Why a shared bug process matters

Without a common loop, every engineer invents intake, triage, and “fix” habits. That burns time on incomplete tickets, mixed-up severity language, and untested patches. This course teaches one end-to-end path so the team ships fewer regressions and closes bugs with Root Cause + Prevention — not just a green PR.

Principles: Reproducibility First (fulfilled by Diagnosis Loop Phases 1–2) · Skills-driven · Root Cause + Prevention · /tdd + /code-review mandatory · Improve quarterly.

Skills chain: /diagnosing-bugs → /tdd → /code-review (plus /triage and grill skills at intake).
