Triage

Bug skill · after report · before diagnosis

Confirm the claim, set Severity and Priority separately, pick a Triage State, and route to human or agent — without starting diagnosis on incomplete tickets.

← Module 2: Triage /grill-with-docs /diagnosing-bugs

Why

Severity is impact; Priority is scheduling. Mixing them burns capacity. Triage also decides whether the ticket is ready for work or needs more info.

When

  • A new bug report lands (or returns from needs-info)
  • Reporter suggested Severity/Priority — triage confirms or changes
  • Category might be enhancement, not bug — route correctly

What triage sets

  • Category — exactly one: bug or enhancement
  • Severity — severity/s1 … severity/s4 (workaround-aware)
  • Priority — priority/p0 … priority/p3
  • State — needs-triage · needs-info · ready-for-agent · ready-for-human · wontfix

Work starts only at ready-for-agent or ready-for-human.

Failure modes

  • Starting /diagnosing-bugs while still needs-info
  • Collapsing Severity and Priority into one P1–P4 scale
  • Guessing missing template fields instead of grilling or returning
