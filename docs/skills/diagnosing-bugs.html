Diagnosis Loop

Bug skill · after triage · before /tdd fix

Six-phase loop that turns a confirmed bug into a root cause and a regression-ready fix path. Reproducibility first — do not patch until you have a red signal.

← Module 3: Diagnosis /triage /tdd

Why

Guess-and-patch creates regressions. The Diagnosis Loop forces a deterministic feedback command, a minimized reproduce, ranked hypotheses, and one-variable instrumentation before the fix.

When

  • Triage State is ready-for-agent or ready-for-human
  • Bug report template is complete (or grilled to completion)
  • You are about to change production code for a defect

Skip only for trivial typos with an obvious red test already in hand — still run a minimal reproduce.

Six phases

  1. Feedback loop — one red-capable, deterministic command
  2. Reproduce + minimise — shrink until every step is load-bearing
  3. Hypothesise — 3–5 ranked, falsifiable hypotheses
  4. Instrument — one variable at a time; tagged probes
  5. Fix + regression test — prefer test-before-fix at a correct seam; hand off to /tdd
  6. Cleanup + post-mortem — green loop, remove debug harnesses, ticket RCA notes

Failure modes

  • Fixing before a reliable red signal
  • Skipping minimise — debugging the whole system at once
  • Jumping to /implement or a bare PR without diagnosis notes
