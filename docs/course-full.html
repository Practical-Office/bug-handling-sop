Bug Handling SOP

Full printable course (~90–120 min). Living SOP: docs/reference/BUG-HANDLING-SOP.md

1

Why good bug processes matter

~15 min

Goals

  • Reduce time-to-resolution
  • Improve reproducibility and root cause analysis
  • Enforce testing and prevent regressions
  • Build Cursor skills muscle memory
  • Measurable improvement in quality and velocity

Principles

Reproducibility first · Skills-driven · Root cause + prevention · TDD + review mandatory · Improve quarterly

2

Reporting & triage

~20 min

Template fields

Title · Environment · Reproduction Steps · Expected · Actual · Logs/Screenshots · Business Impact

Severity

P1 triage 1h · P2 4h · P3 minor · P4 cosmetic

3

Diagnosis & fixing

~30–40 min

Skills

/diagnosing-bugs then /tdd

Loop

Reproduce → Minimize → Hypothesize → Instrument → Fix → Regression test

Workflow

  1. In Progress → diagnose → minimal repro → TDD fix → tests → document findings
4

Review, closure & prevention

~20–25 min

  1. PR references ticket
  2. /code-review
  3. Ticket: Root Cause + Prevention
  4. QA verifies → close

Metrics

Time to fix by severity · % with repro steps · % with regression tests · recurring rate · team feedback
