Setup
Pre-requisite · ~15 min

Skills setup

Install and configure Matt Pocock Cursor skills before Module 1. This is a one-time Setup Pre-req — not a numbered content module.

Learning objectives

What you will leave able to do

  • Install the Matt Pocock skills pack into Cursor (and optionally other agents).
  • Run /setup-matt-pocock-skills once per repo and know what files it creates.
  • Know which skills are user-invoked vs model-invoked for bug work.

Hard truth

Modules 1–5 assume these skills are available. Skipping Setup means you will invent process instead of using the shared loop — and your Evidence Pack will not match how the team actually works.

Step 1

Install Matt Pocock skills

Preferred path (skills.sh — editable copies in the project):

npx skills@latest add mattpocock/skills

During install:

  1. Select the skills you need (at minimum the bug-handling set below).
  2. Select Cursor as a target agent.
  3. Ensure you select /setup-matt-pocock-skills.

Minimum bug-handling set: diagnosing-bugs, tdd, triage, code-review, grill-me / grill-with-docs, and setup-matt-pocock-skills.

Optional Claude Code plugin path: claude plugin marketplace add mattpocock/skills then claude plugin install mattpocock-skills@mattpocock — still run /setup-matt-pocock-skills once per repo afterward.

Step 2

Run /setup-matt-pocock-skills

This skill is prompt-driven, not a silent script. It will:

  1. Explore the repo (git remote, AGENTS.md / CLAUDE.md, existing docs/agents/, triage skill presence).
  2. Ask (with recommended defaults):
    • Issue tracker — GitHub / GitLab / local markdown / other
    • Triage label vocabulary — defaults: needs-triage, needs-info, ready-for-agent, ready-for-human, wontfix
    • Domain docs layout — usually single-context CONTEXT.md + docs/adr/
  3. Write / update:
    • docs/agents/issue-tracker.md
    • docs/agents/triage-labels.md (if triage installed)
    • docs/agents/domain.md
    • An ## Agent skills block in AGENTS.md or CLAUDE.md

Team default for Practical AI repos: GitHub Issues via gh, with triage state labels aligned to Matt’s roles. Severity (severity/s1–severity/s4) and Priority (priority/p0–priority/p3) sit alongside those states — see triage-labels.md and Module 2.

On this training repo, confirm docs/agents/issue-tracker.md and docs/agents/triage-labels.md already match the Living SOP — re-run setup only if something is missing or wrong.

Step 3

Team boards (GitHub Projects)

Practical AI uses two org-level boards for daily work. Product bugs are filed on the owning repo (e.g. Book-IQ/bookiqv1-rc), then added to the Bug Report board when useful.

  • Bug Report — bug intake: Needs Triage → Needs Info → Ready → In Progress → Done. Class examples use the Example field; filter them out of your Ready queue.
  • Team Work — per-teammate BML, Update, and Bug work. Set Assignees + Ticket Type on every card. Use the Team roster table (group by Assignees) or My work view (assignee:@me). View setup guide →

Do not file production incidents only on this training repo. See issue-tracker.md.

Security reporting (product repos)

Product repos must define a private vulnerability path — usually root SECURITY.md plus GitHub private reporting. This training repo includes a minimal stub; do not paste exploit details in public issues. Module 2 covers triage routing for security bugs.
