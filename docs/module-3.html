Diagnosis & fixing with skills

~30–40 min (may split later)

Hard truth

Diagnosis without a minimized reproduction is guessing. TDD without a failing test first is theater.

Mandatory skills

  • /diagnosing-bugs — core diagnostic loop
  • /tdd — red → green → refactor for every fix

Diagnostic loop (/diagnosing-bugs)

  1. Reproduce — confirm the report on a known environment.
  2. Minimize — smallest test case that still fails.
  3. Hypothesize — name a falsifiable cause.
  4. Instrument — add logging or probes that can disprove the hypothesis.
  5. Fix — change only what the evidence supports.
  6. Regression test — lock the behavior so it cannot silently return.

Step-by-step workflow

  1. Acknowledge ticket; set status to In Progress.
  2. Run /diagnosing-bugs.
  3. Create a minimal reproduction if needed.
  4. Implement the fix using /tdd.
  5. Add or update relevant tests.
  6. Document findings in the ticket (partial notes OK before PR).

Skills setup (one-time)

  1. Install Matt Pocock skills: npx skills@latest add mattpocock/skills
  2. Run /setup-matt-pocock-skills in Cursor (if available).
  3. Master: diagnosing-bugs, tdd, triage, code-review, grill-me / grill-with-docs.

Paired exercise

With a partner (or agent), pick a sample bug from your product backlog. Run the diagnostic loop out loud through hypothesize before writing any fix. Note where you would add the first failing test.
