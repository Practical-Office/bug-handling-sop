# Security Policy

This repository is a **Bug Handling SOP training course**, not a production application. It does not run customer workloads.

## Reporting a vulnerability

- **In this training repo:** open a GitHub issue with a high-level description only — no exploit payloads, secrets, or step-by-step attack instructions in public threads.
- **In Practical AI product repositories:** use that repo’s private vulnerability reporting (GitHub **Report a vulnerability** / Security Advisories) and follow its `SECURITY.md` when present.
- **Do not** file public bugs with unfixed vulnerability details.

## Course reference

The Published Course teaches the private advisory path in Module 2 (Triage). See the Living SOP §6.6 and `.github/ISSUE_TEMPLATE/bug-report.yml`.
